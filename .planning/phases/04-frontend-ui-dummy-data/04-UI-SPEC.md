# Phase 4 — UI Design Contract

> Visual and interaction contract for the FairyWeave frontend. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (pure Tailwind v4 utility classes) |
| Icon library | none (emoji for theme chips, inline SVG for nav arrows) |
| Font | Jua (Google Fonts) -- rounded Korean storybook font |

**Font loading:** Import via `next/font/google` in `layout.tsx`. Apply as CSS variable `--font-jua` on `<html>`. Body and all headings use Jua. Fallback stack: `'Jua', 'Apple SD Gothic Neo', sans-serif`.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip internal padding-x |
| sm | 8px | Compact element spacing, chip gaps |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section inner padding, card body padding |
| xl | 32px | Layout gaps between major blocks |
| 2xl | 48px | Section vertical padding (hero, gallery, footer CTA) |
| 3xl | 64px | Page top/bottom margin |

Exceptions: none

**Implementation:** Use Tailwind spacing utilities (`p-4` = 16px, `gap-6` = 24px, etc.). No custom spacing tokens needed -- Tailwind's default 4px scale covers all values.

---

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 (regular) | 1.6 | `text-base font-normal leading-relaxed` |
| Label | 14px | 600 (semibold) | 1.4 | `text-sm font-semibold leading-snug` |
| Heading | 24px | 700 (bold) | 1.3 | `text-2xl font-bold` |
| Display | 32px | 700 (bold) | 1.2 | `text-3xl font-bold leading-tight` |

**Weights used:** 400 (body, placeholder text) and 700 (headings, buttons, labels). Jua ships a single weight (400) that reads as bold -- supplement with `font-bold` for display/heading where extra emphasis is needed. If Jua's single weight proves insufficient for visual hierarchy, fall back to weight contrast via size and color alone.

**Korean text note:** Line height 1.6 for body accommodates Korean syllable blocks and descenders.

## Page Contracts

### Home (`/`)

**Hero Section:**
- Full-width, `py-12` (48px) vertical padding
- Display heading: "세상에 단 하나뿐인 동화책" at 32px bold
- Subheading: "우리 아이만의 이야기를 AI가 만들어 드려요" at 16px regular, secondary text color
- Primary CTA: "내 아이 동화책 만들기" -- coral accent, white text, `rounded-2xl`, `px-8 py-4`
- CTA links to `/create`

**Gallery Section:**
- Heading: "인기 동화책" at 24px bold
- Grid: `grid-cols-2 sm:grid-cols-3 gap-4` (16px gaps)
- 5 dummy book cards

**Gallery Card:**
- Soft pink background (`#FDE8E8`), `rounded-2xl`, overflow hidden
- Cover image: aspect-ratio 1/1, object-cover, fills top portion
- Card body: `p-4` (16px padding)
- Title: 16px bold, primary text color, single line truncate
- Theme badge: 14px, matching theme chip color, `rounded-full`, `px-2 py-0.5`
- Entire card is clickable, links to `/book/[dummyId]`
- Hover: subtle scale `hover:scale-[1.02]` + shadow transition

**Bottom CTA Section:**
- Background: soft pink
- Text: "직접 만들어 보세요" at 24px bold
- CTA button: same style as hero CTA
- Links to `/create`

### Create Form (`/create`)

**Layout:**
- Centered container, `max-w-lg mx-auto`, `py-12`
- Form card: soft pink background, `rounded-2xl`, `p-6`

**Fields (top to bottom):**

1. **아이 이름** (child name) -- text input, required
   - Label: "아이 이름" at 14px semibold
   - Input: full width, `rounded-xl`, `px-4 py-3`, cream background, 1px border `#E0D6CC`
   - Placeholder: "주인공 이름을 입력해 주세요"

2. **나이** (age) -- dropdown, required
   - Label: "나이" at 14px semibold
   - Select: full width, same styling as text input
   - Options: 1세 ~ 10세
   - Default: placeholder "나이를 선택해 주세요"

3. **테마** (theme) -- chip grid, required
   - Label: "테마" at 14px semibold
   - Chips: horizontal flex wrap, `gap-2` (8px)
   - Each chip: `rounded-full`, `px-4 py-2`, cream background, 1px border `#E0D6CC`
   - Chip content: emoji + Korean label (e.g. "🦕 공룡")
   - Selected chip: accent coral border + light coral background `#FFF0EB`
   - Available chips: 🦕공룡, 🚀우주, 🧙마법, 🌊바다, 🌲숲, ✏️직접 입력
   - "직접 입력" chip selected: reveals text input below chips

4. **교훈** (moral/lesson) -- textarea, optional
   - Label: "스토리 방향 / 교훈" at 14px semibold
   - Textarea: 3 rows, full width, same border styling
   - Placeholder: "예: 형제간의 우애, 편식 극복"

**Validation (D-19, FORM-03):**
- Client-side on submit
- Missing required field: red border (`#E55B5B`) + inline error text below field at 14px
- Error messages: "이름을 입력해 주세요", "나이를 선택해 주세요", "테마를 선택해 주세요"

**Submit button:**
- "동화책 만들기" -- full width, coral accent, white text, `rounded-2xl`, `py-4`
- Disabled state: opacity 50%, cursor not-allowed (while submitting)

### Loading State (AI-04, D-21)

**Appears after form submit, replaces form content:**
- Centered layout within the same page
- Skeleton card: soft pink, `rounded-2xl`, `animate-pulse` on image placeholder area
- Progress bar: full width, 8px height, `rounded-full`, cream track, coral fill
- Step text below bar at 16px regular:
  - "스토리 쓰는 중..." (0-30%)
  - "삽화 그리는 중 (3/16)..." (30-90%, counter updates per image)
  - "거의 다 되었어요!" (90-100%)
- No cancel button (generation is fast enough)

### Book Preview (`/book/[id]` -- Step 1: Preview)

**Image area (upper 2/3 of viewport):**
- Full-width image, `rounded-2xl`, object-cover
- `aspect-[3/4]` ratio
- Max height: `max-h-[60vh]`

**Text + navigation area (lower 1/3):**
- Story text: 16px regular, primary text color, `leading-relaxed`
- Page indicator: "3 / 16" at 14px, secondary text color, centered
- Navigation arrows: inline SVG chevrons, 32px touch target with 8px padding
  - Left arrow: `opacity-30` on page 1 (disabled)
  - Right arrow: `opacity-30` on page 16 (disabled)
- Arrow buttons: `rounded-full`, cream background, positioned at horizontal edges

**Keyboard:** Left/Right arrow keys navigate pages.

**"주문하기" button below preview:**
- Full width, coral accent, white text, `rounded-2xl`, `py-4`
- Label: "이 동화책 주문하기"
- Transitions to Step 2 (order form)

### Order Form (`/book/[id]` -- Step 2: Order)

**Layout:** Same container as preview, step transition via React state.

**Heading:** "배송 정보" at 24px bold

**Fields (matching backend `POST /api/sweetbook/orders` body):**

1. **수령인 이름** (recipientName) -- text input, required
   - Placeholder: "받으실 분 이름"

2. **연락처** (recipientPhone) -- tel input, required
   - Placeholder: "010-1234-5678"

3. **우편번호** (postalCode) -- text input, required
   - Placeholder: "12345"

4. **주소** (address1) -- text input, required
   - Placeholder: "서울시 강남구 테헤란로 123"

5. **상세주소** (address2) -- text input, optional
   - Placeholder: "101동 202호"

6. **배송 메모** (shippingMemo) -- text input, optional
   - Placeholder: "부재 시 경비실에 맡겨 주세요"

**Validation:** Same red-border pattern as create form. Required field errors in Korean.

**Submit button:**
- "주문하기" -- coral accent, white text, full width, `rounded-2xl`, `py-4`
- Loading state: spinner icon + "주문 중..." text, disabled

**Back button:** text-only link "돌아가기" above form, navigates back to Step 1.

### Order Complete (`/book/[id]` -- Step 3: Complete, D-23)

**Layout:** Centered, `text-center`

**Content:**
- Success icon: large checkmark in mint circle, 64px
- Heading: "주문이 완료되었어요!" at 24px bold
- Order ID: "주문번호: {orderUid}" at 14px, secondary text, `font-mono`
- Summary card (soft pink, `rounded-2xl`, `p-4`):
  - Book title at 16px bold
  - Recipient name + address at 14px regular

**Buttons:**
- Primary: "또 만들기" -- coral accent, white text, `rounded-2xl`, links to `/create`
- Secondary: "메인으로" -- outline style (coral border, coral text, cream background), `rounded-2xl`, links to `/`
- Button row: horizontal flex, `gap-4`

## Error States (D-22)

**Inline error (form validation):**
- Red border (`#E55B5B`) on the offending field
- Error message text below field: 14px, `#E55B5B`, appears on submit attempt

**API error (generation failure, order failure):**
- Full-width alert banner within the content area
- Soft red background `#FDE8E8` with red left border (4px `#E55B5B`)
- Heading: "앗, 문제가 발생했어요" at 16px bold
- Body: error description at 14px regular
- "다시 시도하기" button: destructive red outline, `rounded-xl`
- Appears in-place (replaces loading state or order form submit area)

## Interaction Patterns

| Pattern | Specification |
|---------|---------------|
| Page transitions | No client-side page transition animations. Standard Next.js navigation. |
| Step transitions (book page) | React state-driven. Steps: preview -> order -> complete. No URL change. |
| Card hover | `transition-transform duration-200 hover:scale-[1.02] hover:shadow-lg` |
| Button press | `active:scale-95 transition-transform duration-100` |
| Theme chip select | Instant border/bg color swap, no animation |
| Slider page change | No slide animation for v1. Instant swap. Keyboard arrow support. |
| Loading pulse | Tailwind `animate-pulse` on skeleton areas |
| Focus states | `focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-[#FF8C6B]` on all interactive elements |
